Template Installation

  1. You'll either need a local server (like XAMPP or MAMP) or remotely hosted server and a domain (Dreamhost, BlueHost and similar).
  2. Create domain, subdomain or simply create new folder on the server where template files will be copied.
  3. If using localhost copy-paste all the files from "Buntington_HTML" folder to the destination folder on the server. Otherwise use FTP client (like FireFTP) to transfer files from "Buntington_HTML" to the remote server.

Template Markup Structure of files

Buntington is made with Twitter Bootstrap and is fully adaptable to any known viewport (screen resolution).

This template has 16 different pages but all of them are using the same base structure. Content is divided into sections and each section uses unique ID. Because of the fact that entire layout depends on Bootstrap framework, all other elements within sections are styled and enhanced with extensible classes and an advanced grid system.

Styles / CSS

It'll make things look right in Browser.

First off, take a look at the HEAD Markup, line #16. You can notice that the template main stylesheet gets loaded last. This is due to the fact that any eventual plugin or framework CSS can be overwritten in our main stylesheet.
Otherwise, whichever correction/improvement needs to be done to either a framework (like Twitter Bootstrap) or a plugin CSS, it can be done in our main stylesheet file. Third party stuff gets updated which means that any changes you made might get lost. 

<link href="style.css" rel="stylesheet" type="text/css"><!-- theme styles -->

// How to work with "style.css"?

In case you are about to change style/layout of a certain element on page you'll definitely have to open and edit "style.css". It's always good idea to use Browser developer tools hence these are capable to point you to the exact line number of a target element within CSS file. I think the very best developer tool is definitely Firebug and my friendly advice is to get it. All of modern Browsers are equipped with developer tools and it's OK to use these as well.

No matter what, once you open "style.css", you'll notice smaller (or larger, of course) chunks of code each one preceded with a comment line. Such structure should help you find target element CSS easier or at least serve as a reminder of what belongs where.

For example, if you want to edit Tabs and/or Pills you gonna search for either tabs or pills by using preferred code editor's FIND function or simply scroll down the file until section found. 

...
/* == bootstrap tabs and pills == */

.nav-tabs > li > a { -moz-border-radius: 0 !important; -webkit-border-radius: 0 !important; border-radius: 0 !important; font-weight: 700; font-size: 14px; }

In certain style definition modify whatever you find suitable, save "style.css", get back to Browser then refresh page to see changes. Of course, CSS knowledge is a must otherwise results might be unpredictable.

Javascript JS used by Buntington

Buntington template uses a couple of jQuery plugins in order to make the final impression a little bit more awesome than average. Just kidding :) No, I'm not. Without javascript any website would probably look boring or so.
Let's see what is needed by our template...
First off, you need to know that all the javascript code is loaded in page footer and that's where you gonna go looking for it. Why footer? Because that's how all other content (including photos) gets loaded sooner and the page doesn't look empty for a couple of seconds (while a ton of JS code is loading). First impression is important, right?

Template Editing Short and smart how-to

Heads up!
Code editor is needed if you want to be able to modify/edit files in this template. So in case you don't have it installed yet, here are some for your consideration:

  • Notepad++ (for Windows)
  • TextMate (for MacOS)
  • Sublime Text (cross platform, for both Win and MacOS)

// How to change link colors?

Open "style.css" and pay your attention to lines #30 and #31. Change HEX value of the color property for both a:link and a:hover pseudo classes then save "style.css".

For example, if I want to change the color of links from red to green (and leave hover color as is), I should modify line #30 only: 

/* == font and links == */
a:link, a:visited { color: #ea5644; text-decoration: none; }
a:hover, a:focus { color: #111; outline: none; }
/* == END font and links == */

Replace #ea5644 with #2cb76b 

/* == font and links == */
a:link, a:visited { color: #2cb76b; text-decoration: none; }
a:hover, a:focus { color: #111; outline: none; }
/* == END font and links == */

// How to replace default logo?

You will need to have your own logo file in PNG format. It should be somewhere around 170x120px but if wider - it won't make any harm.

Rename your own file to "site-logo.png" then copy-paste to "Buntington_HTML/img" folder. Confirm overwrite when asked!

Get back to Browser, refresh page and voila - your own logo is there.

// How to change the address on GoogleMap?

Each and every GMap is added on page by using the following HTML syntax: 

<div id="k-contact-map" class="clearfix"><!-- map -->
    <div 
    id="g-map-1" 
    class="map" 
    data-gmaptitle="Contact" 
    data-gmapzoom="15" 
    data-gmaplon="-76.422377" 
    data-gmaplat="43.314594" 
    data-gmapmarker="" 
    data-cname="Buntington Public Schools" 
    data-caddress="115 W Broadway" 
    data-ccity="Fulton" 
    data-cstate="NY" 
    data-czip="13069" 
    data-ccountry="USA">
    </div>
</div>

Where...

  • id="g-map-NN" - represents GoogleMap element ID. As you probably know element ID must be unique!
  • class="map" - class "map" is a MUST, see "style.css", line #429
  • data-gmaptitle="Map Title" - should be a title of your map.
  • data-gmapzoom="NN" - map zoom level (NN should range from 1 to 19)
  • data-gmaplon="NN.NNNN" - map/address longitude
  • data-gmaplat="NN.NNNN" - map/address latitude
  • data-gmapmarker="http://full_path_to_a_custom_marker_file" - path to a custom marker file
  • data-cname="Map window title" - title of a map window
  • data-caddress="Address" - address on a map window
  • data-ccity="City" - city on a map window
  • data-cstate="State" - state on a map window
  • data-czip="ZIP code" - zip code on a map window
  • data-ccountry="Country" - country on a map window

No further javascript code modifications are required, however, if you really need to change something open "Buntington_HTML/js/theme.js" and see GMaps code between lines #63 and #149.

// How to use FontAwesome icons?

FontAwesome contains over 430 icons that can be used with this template. The syntax is quite easy: 

<i class="fa fa-ICON_NAME"></i>

ICON_NAME should be replaced with the real icon name, for example "camera-retro".

FontAwesome examples page has more cool usage examples. You will also need a page with the list of all icons at one place.

// How to use different Google web font?

There's plenty of free web fonts available from Google webfonts repository and you can load any other instead of template's default.

Take a look at the HEAD markup of ANY template file, all of them include Open Sans (line #10). It means that if you want to change the font, you'll have to do it across all template files!

So let's say I want to use Roboto instead of Open Sans. In that case I would replace: 

<link href="http://fonts.googleapis.com/css?family=Open+Sans:400,300,300italic,700,800" rel="stylesheet" type="text/css"><!-- Google web fonts -->

...with the following: 

<link href="http://fonts.googleapis.com/css?family=Roboto:100,300,400,300italic,500,700,900" rel="stylesheet" type="text/css"><!-- Google web fonts -->

Now, since all of my template files include new font family, I have to update main stylesheet as well.

Open "Buntington_HTML/style.css" for edit, go to line #28 and replace: 

body { font-family: 'Open Sans', 'Myriad Pro', 'Helvetica',  sans-serif; font-size: 16px; font-weight: 300; line-height: 26px; color: #777; }

...with: 

body { font-family: 'Roboto', 'Myriad Pro', 'Helvetica',  sans-serif; font-size: 16px; font-weight: 300; line-height: 26px; color: #777; }

Can I use more than one font family? Roboto for H1-H6 elements and Open Sans for text?

Sure you do! Simply load both fonts in HEAD markup: 

<link href='http://fonts.googleapis.com/css?family=Open+Sans:400,600,700,800,400italic,600italic,700italic,800italic|Roboto:400,300,500,700,300italic,400italic' rel='stylesheet' type='text/css' /><!-- Google web fonts -->

...and then update "style.css" accordingly: 

h1, h2, h3, h4, h5, h6 { font-family: 'Roboto', sans-serif; font-weight: 700; color: #636363; }
